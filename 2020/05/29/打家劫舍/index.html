打家劫舍

题目表述：

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，

如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。
给定一个代表每个房屋存放金额的非负整数数组，计算你 不触动警报装置的情况下 ，一夜之内能够偷窃到的最高金额。

示例 1:
输入: [1,2,3,1]
输出: 4
解释: 偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = 3)。
     偷窃到的最高金额 = 1 + 3 = 4 。
示例 2:
输入: [2,7,9,3,1]
输出: 12
解释: 偷窃 1 号房屋 (金额 = 2), 偷窃 3 号房屋 (金额 = 9)，接着偷窃 5 号房屋 (金额 = 1)。
     偷窃到的最高金额 = 2 + 9 + 1 = 12 。

个人总结:

  • 其实本题一看应该是动态规划问题,因为是求的是最高金额,但这道题的动态规划的状态转移方程需要如何编写呢,状态转移方式是动态规划问题中最难生成的。
  • 这道题其实我自己考虑的时候也在想如何写出状态转移方式,我的心路历程是考虑每一个数的最值问题,我只是想到了此处,想到是取到 dp[i(当前数据) - 2] + nums[i]，后面的没有想出来。。。
  • 其实看了题解之后,发现只是差临门一脚了,这种问题后面还是要多多联系,有了思路可能就好了吧。就相当于原先看到一道题并不知道需要使用什么方法来求解,现在是看到一道题知道需要使用什么方式来求解了。(说的有点远了，回归主题)
  • 状态转移方式就是当前的最大值是：当前值的前两个的最大值 + 当前值 与 当前值的前一个最大值进行比较,去这两个中的最大值为当前的最大值,状态转移方程为：
        dp[i] = max(dp[i-2] + nums[i], dp[i-1])
  • 得到方程了,在考虑边界的情况,由于需要是前两个,所以数组的前两个值单独计算一下即可。以下为代码
   class SolutionAL:
    def rob(self, nums: List[int]) -> int:
        if not nums:
            return 0
        lenght = len(nums)
        if lenght == 1:
            return nums[0]
        dp = [0] * lenght
        dp[0], dp[1] = nums[0], max(nums[0], nums[1])
        for i in range(2, lenght):
            dp[i] = max(dp[i-2] + nums[i], dp[i-1])
        return dp[-1]
  • 以上方式的时间复杂度为O(n)，空间复杂度也为O(n)，由于是创建了一个数组来保存每一位的最大值。
  • 其实还可以进行优化为空间复杂度为O(1),经过观察发现当前值只是跟前两个和前一个元素有关,那我们就可以进行声明两个变量重复的进行替换。类似于斐波那契数列。
  • 可以理解成在数组前面增加了两个数,这两个数均为0,相当于动态规划的i-2 和 i-1 这样只需要两个变量就可以了。这种解法也是通过状态转移方程生成之后转化而来的。
class SolutionAL:
    def rob(self, nums: List[int]) -> int:
        a, b = 0, 0
        c = 0
        for num in nums:
            c = max(num + a, b)
            a = b
            b = c
        return c
  • 可以看出代码相当的简洁。。。
  • 其实往往复杂的问题都是从简单的问题演化而来的。
