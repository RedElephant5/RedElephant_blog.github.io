2020-09-29
  • Pycharm

直觉判断应该跟jdk有关，想起了原先在进行Pycharm破解的时候安装过jar包，应该跟那个jar包有关系。在删除了那个jar包之后发现报错信息不一样了！感觉思路是对的。

OpenJDK 64-Bit Server VM warning: Option UseConcMarkSweepGC was deprecated in version 9.0 and will likely be removed in a future release.
Error opening zip file or JAR manifest missing : /usr/share/pycharm-2018.3.7/bin/JetbrainsCrack-4.2-release-enc.jar
Error occurred during initialization of VM
agent library failed to init: instrument

但为什么会直接定位到原先的那个jar包呢？通过再网上查找资料应该是pycharm在安装的时候会查找原先的配置JetBrains 通过 find . -name 'JetBrains' 查找到在家目录下有三个隐藏文件存在

./home/xxx/.local/share/JetBrains
./home/xxx/.config/JetBrains
./home/xxx/.cache/JetBrains

直接删除这三个文件在打开的时候还是会出现第一个报错的情况。 所以查找是否有具体的配置文件,看配置文件中是否有引用原先的jar包，在.config/JetBrains/PyCharm2020.2中发现pycharm64.vmoptions文件 其中有引用原先jar的配置。

-javaagent:/usr/share/pycharm-2018.3.7/bin/JetbrainsCrack-4.2-release-enc.jar

把这行配置进行删除，保存。再打开Pycharm2020.2成功打开。
